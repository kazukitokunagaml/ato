# コントリビューションガイドライン

## 行動規範

本プロジェクトに興味をお持ちくださり、ありがとうございます。本プロジェクトでは、[Code of Conduct](CODE_OF_CONDUCT.md)を採用しています。参加することで、この規範を遵守することに同意したものとみなされます。

## コントリビューションの方法

### バグ報告

バグを発見した場合は、以下の情報を含めてIssueを作成してください。

- バグの簡潔な説明
- 再現手順
- 期待される動作
- 実際の動作
- スクリーンショット（該当する場合）
- 環境情報（OS、ブラウザ、Node.jsバージョンなど）

### 機能提案

新機能のアイデアがある場合は、Issueを作成して以下を記載してください。

- 機能の説明
- なぜこの機能が必要か
- 可能であれば、使用例やモックアップ

### プルリクエスト

1. **ブランチの作成**
   ```bash
   git checkout -b ユーザー名/対応内容(例: saigo/setup-nextjs)
   ```

2. **コミット**
   ```bash
   git add .
   git commit -m 作業内容
   ```

3. **プッシュとPR作成**
   ```bash
   git push origin ブランチ名
   ```

その後、GitHubでプルリクエストを作成してください。

### レビュー

1. プルリクエストを作成すると、メンテナーがレビューします
2. 変更が必要な場合、フィードバックが提供されます
3. 修正後、再度レビューされます
4. 承認されると、メインブランチにマージされます

## ライセンス

このプロジェクトにコントリビューションすることで、あなたのコントリビューションがMITライセンスの下でライセンスされることに同意したものとみなされます。
