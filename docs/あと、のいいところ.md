  ---
  ATO（あと、）の良いところ

  コンセプト・思想が一貫している

  **「無為自然」**をコンセプトに、「紙とペンのような自然な書き心地」を目指したエディタ。名前の由来も「道教 > 道 > TAO > ATO」と思想から一本筋が通っている。キャッチコピーは「只々、自然に書くためのアプリ。」

  ---
  解決する4つの壁が明確

  既存ツールが抱える問題を4つの壁として定義し、すべてに対して具体的な解決策がある。

  ┌────────────────┬──────────────────────────────────────────────────────────────────────┐
  │       壁       │                                解決策                                │
  ├────────────────┼──────────────────────────────────────────────────────────────────────┤
  │ 場所の壁       │ マルチデバイス対応・オフライン動作・PWA                              │
  ├────────────────┼──────────────────────────────────────────────────────────────────────┤
  │ 疲れるという壁 │ 認知負荷ゼロのUI設計・直感的な操作                                   │
  ├────────────────┼──────────────────────────────────────────────────────────────────────┤
  │ サービスの壁   │ ZIP/フォルダのインポート/エクスポート・標準技術採用                  │
  ├────────────────┼──────────────────────────────────────────────────────────────────────┤
  │ 機能の壁       │ バージョン管理・AIエージェント対応（将来）をノーコード知識で使えるUI │
  └────────────────┴──────────────────────────────────────────────────────────────────────┘

  ---
  ローカルファーストで本気のオフライン対応

  - **CRDT（Yjs）**による自動マージ同期 — 競合が起きても自動解決
  - オフライン中の操作はローカルに蓄積し、オンライン復帰時に自動同期
  - PWA対応でブラウザ再起動後もオフラインで起動可能
  - リトライは指数バックオフで最大3回、失敗後もデータは失わない

  ---
  エクスポートが強い（データの自由）

  - ZIPエクスポートはオフラインでも動作（サーバー通信なし）
  - フォルダのインポートも対応（失敗時はロールバック保証）
  - Markdownファイルを標準形式として採用 → 他ツールへの移行が容易

  ---
  セキュリティ設計が丁寧

  - JWTはRS256非対称鍵 + HttpOnly Cookie + SameSite=Strict（CSRFトークン不要）
  - **RLS（行レベルセキュリティ）**でユーザー間のデータを完全分離
  - DB側はNeon AES-256 at-rest encryption
  - レート制限・XSS対策・SQLインジェクション対策・ブルートフォース対策まで網羅
  - 「防御できない脅威」も正直に明記している誠実な設計

  ---
  技術スタック選定の理由が合理的

  ┌────────────────────┬──────────────────────────────────────────────────────────────┐
  │        技術        │                             理由                             │
  ├────────────────────┼──────────────────────────────────────────────────────────────┤
  │ Cloudflare Workers │ 低運用コストでスケール可能、CRDT同期の制御が厳密にできる     │
  ├────────────────────┼──────────────────────────────────────────────────────────────┤
  │ Neon (PostgreSQL)  │ 標準技術でロックイン回避、バックアップ・バージョン管理が容易 │
  ├────────────────────┼──────────────────────────────────────────────────────────────┤
  │ pnpmモノレポ       │ APIの型をフロントに直接共有できる（型安全なAPIクライアント） │
  ├────────────────────┼──────────────────────────────────────────────────────────────┤
  │ OpenAPI            │ スキーマ駆動開発で仕様と実装のズレを防ぐ                     │
  └────────────────────┴──────────────────────────────────────────────────────────────┘

  ---
  UIデザインへのこだわり

  - 和のカラーパレット（墨・白金・露草・翠・山吹・京緋）
  - アクセシビリティ対応（コントラスト比 4.5:1以上、prefers-reduced-motion対応）
  - フォントはユニバーサルデザインの「BIZ UDゴシック」を標準採用
  - 利き手によるパネル配置変更（将来対応）など、細部まで設計済み

  ---
  まとめ

  既存ツールのいいとこ取り（NotionのUX × ObsidianのMarkdown × VSCodeの拡張性）を「自然さを損なわない」という1本の軸で再設計した、コンセプトと設計の一貫性が特に強みのプロダクトです。


言語化
「Notionに不満を持つ書き手に向けた、データが自分のものになるMarkdownエディタ。PCでもスマホでも、同じように書ける。」

  「アイデアは電車の中で浮かぶのに、ちゃんと書けるのはPCの前だけ」                                                                                                                                                                                                          
                                                                                                                                                                                                                                                                          
  この体験に刺さる言葉を考えると：

  ---
  一言で言うなら

  noteに書く前の、思考の場所。

  ---
  もう少し説明するなら

  スマホで書いたメモが、帰ってPCを開いたらそのまま続きが書ける。
  下書きから清書まで、noteに貼り付けるだけ。

  ---
  ポジショニングとしては「あと、」→ noteという一方向の流れが自然で：

  思いつく → 下書き → 清書     →   投稿・公開
          [  あと、  ]               [  note  ]

  この「noteの前段階を全部ここで」という立ち位置が一番クリアだと思います。「データが〜」より「書きたい気持ちが途切れない」という情緒の方がnoteライターには刺さるはずです。
