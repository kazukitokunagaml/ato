## 1. ユーザー分析

### 1-1. noteで反応してくれた人たちは何者か

エディター開発を宣言した記事に「スキ」したユーザーを属性別に見ると、以下の傾向がある。

| 属性 | 特徴 | ATOへの期待 |
|---|---|---|
| **個人開発者**（最多） | テック知識あり、noteで発信、ツールへのこだわりが強い | 拡張性・Markdown・AI連携 |
| **エッセイイスト** | 書く行為そのものを大切にする、文体や環境に敏感 | 書くことへの没入感・UIの美しさ |
| **小説家** | 執筆環境にこだわるタイプが多い | 集中・疲れない・データを手元に置きたい |
| **AI勉強中** | 新しいツールに積極的、AIと執筆の組み合わせに興味 | 将来のAIエージェント機能 |

**共通項**：
- 「書くこと」を真剣に考えている（ライトユーザーではない）
- 既存のツールに不満を持ち、より良いものを探している
- 自分が使うツールに対してこだわりと審美眼がある

### 1-2. 最も刺さる人物像（コアペルソナ）

**「テック感のある文系クリエイター」**

- noteで定期的に発信している（エッセイ・ブログ・思考整理・開発日記）
- PCとスマホ両方で書いているが、体験が分断されていることに不満を感じている
- NotionやObsidianを試したが、何かが違うと感じて離脱した経験がある
- Gitは「聞いたことがある」程度。使いたいが勉強したくない
- データを自分で管理したいという意識はあるが、具体的な方法を知らない
- ツールが好きで、良いものを見つけたら積極的に紹介する

**刺さらない人（スコープ外）**：
- ガチなエンジニア → VSCode/Obsidianで十分
- ライトユーザー → メモアプリやGoogle Keepで十分
- チーム利用層 → 現時点で共同編集がないため弱い

## 2. 競合ポジショニング

### 2-1. 競合の問題点まとめ

| サービス | 主な問題点 | 離脱要因 |
|---|---|---|
| Obsidian | UIが非直感的で学習コストが高い | 使いこなすまでに本業の時間を奪われる |
| iA Writer | OS別の高額課金、スマホでAI・構文機能が使えない | 機能と値段が見合わない |
| Ulysses | 機能が少なすぎる（Google Keepと差がわからない） | 薄味すぎて乗り換える理由がない |
| Notion | 独自形式でロックイン、AI連携が弱い、エクスポートが重い | MCPの台頭に乗れていない |
| VSCode | スマホに最適化されていない、執筆に不要な機能が多い | 非エンジニアどころかエンジニアでもハードル高い |
| Nola/NOVEWRITE | 小説特化のプラットフォームでエディターではない | 汎用ツールを求めるユーザーには合わない |

### 2-2. ポジショニングマップ

```
              ← 学習コスト高・使いづらい                  使いやすい・直感的 →

高機能・      VSCode ●                    Obsidian ●
拡張性が
ある
    ↑
    |                                               ★ あと、（目指す位置）
    |
    ↓

低機能・                      Notion ●  iA Writer ●
単機能                        Ulysses ●
                低機能・
                単機能
```

**ATOが狙うポジション**：高機能 × 低学習コスト × スマホ対応

このポジションに競合が存在しない。ObsidianはMarkdownと拡張性があるがUIが複雑。NotionはUIが優れているが拡張性に限界がある。ATOはその両方の良いとこ取りができる唯一の位置に立てる。

### 2-3. 各競合ユーザーの離脱先としてのATO

| 離脱元 | 離脱理由 | ATOへの移行ストーリー |
|---|---|---|
| Notion離脱者 | 独自形式のロックインが嫌、AIと連携しづらい | Markdownで書いてZIPで持ち出せる、将来のAI連携も見据えた設計 |
| Obsidian離脱者 | スマホ操作がしんどい、学習コストが高い | スマホでも同じ体験、セットアップ不要 |
| iA Writer離脱者 | 高額課金、スマホで機能制限される | (将来的な価格設計次第で訴求可能) |
| VSCode執筆民 | スマホ対応がない、執筆に不要な機能が多い | VSCodeの拡張性思想を、執筆に特化した形で実現 |

## 3. 強みの言語化

### 3-1. 競合に対して明確に優位な強み（TOP3）

#### ① データの自由（一番のキラーフィーチャー）

> **「あなたのデータは、あなたのもの。いつでも出せて、どこにでも持っていける。」**

- Markdown（業界標準形式）を採用 → 将来どのツールにも移行できる
- オフラインでもZIPエクスポートが動作（サーバー通信なし）
- フォルダのインポートにも対応（失敗時はロールバック保証）
- Notionのロックインに不満を持つユーザーに直撃する訴求

Notionを使っている人に向けた一言：「データを人質に取られていると感じたことはないですか？」

#### ② どこでも、途切れない（最も共感を得やすい問題解決）

> **「電車でスマホに書いたものを、PCで続きから書ける。ただそれだけのことを、ちゃんと実現した。」**

- CRDT（Yjs）によるオフライン動作 + 自動マージ同期
- スマホ・PC・タブレットで同じ書き心地
- オフライン中も書いていた内容は、通信復帰後に自動で同期
- 「移動中にスマホで書き始めて、家のPCで続きを書く」という日常的な不満を解消

#### ③ 機能は強力、でも勉強いらない（将来のAI・バージョン管理に向けた布石）

> **「Gitを知らなくても、バージョン管理ができる。AIを使うのに、プログラミングはいらない。（予定）」**

- 執筆者が本当に必要な機能を、学習コストゼロで使えるUIで提供する
- 現在は基盤を整備中。将来的なAI・バージョン管理対応は、既にロードマップに組み込まれている
- VSCodeとnoteの間にいる「テック感のある文系」層に刺さる

### 3-2. 感情的な強み（ブランドとして）

> **「只々、自然に書くためのアプリ。」**

- コンセプト「無為自然」の一本筋が通っている
- 名前の由来（道教 > 道 > TAO > ATO > あと、）に物語がある
- 和のカラーパレット（墨・露草・翠など）と余白設計
- 「削りすぎた美しさ」ではなく「迷わせない美しさ」を目指している

この思想的な背景は、noteユーザーが好む「作者の人間性が見える」コンテンツと相性がいい。

### 3-3. 一言で言える強み（エレベーターピッチ）

> **「NotionよりMarkdownが自由で、ObsidianよりUIが直感的で、スマホでも同じ体験で書ける、noteユーザーのためのエディタ。」**

## 4. マーケティング計画

### フェーズ概要

| フェーズ | 期間（目安） | 目標 | 主な施策 |
|---|---|---|---|
| Phase 1：信頼構築 | 現在〜1ヶ月 | ファンを作る | note記事・開発日記 |
| Phase 2：獲得 | 1〜3ヶ月 | 初期ユーザーを集める | LP整備・note展開 |
| Phase 3：定着 | 3〜6ヶ月 | 使い続けてもらう | オンボーディング改善・口コミ |
| Phase 4：収益化 | 6ヶ月以降 | 持続可能な運営 | プレミアム機能・有料プラン |

---

### Phase 1：信頼構築（今すぐやること）

**目標**：「この人が作るツールなら使いたい」と思ってもらう

ATOのターゲット（noteユーザー）は、ツールよりも**作者を信頼してから使う**。開発日記形式で、作者の人間性と問題意識を先に届ける。

#### note記事の優先順位（記事案）

| 優先度 | タイトル案 | 狙い |
|---|---|---|
| 🔴 最優先 | 「Notionに限界を感じて、自分でエディタを作った話」 | Notion離脱層への直球訴求 |
| 🔴 最優先 | 「スマホで書いたメモがPCと同期されないストレスを、ずっと抱えていた」 | 共感から入る感情訴求 |
| 🟡 次点 | 「私が試した6つの執筆ツールの正直な感想」 | 競合比較で検索流入も狙える |
| 🟡 次点 | 「データを自分で持ちたい、と思ったら作るしかなかった」 | データ主権の価値観共有 |
| 🟢 任意 | 「なぜ名前が『あと、』なのか」 | ブランド・思想の深掘り |

#### 記事の書き方ポイント

- 「私も同じ問題を抱えていた」という共感から始める
- ツールの宣伝ではなく、自分の体験談として書く
- 最後に「試してみませんか」と誘導する（押しつけない）
- 読者が「それ私も思ってた」と感じたら、そのまま初期ユーザーになる

---

### Phase 2：獲得（LP整備 + note展開）

**目標**：記事を読んだ人がATOを試せる状態を作る

#### ランディングページの必須要素

Phase 1の記事からLPに誘導する前に、LPで以下を3行以内で説明できるようにする。

```
[誰のためのツールか]
noteで書いている人のための、下書き・思考整理エディタ。

[何が違うのか]
Markdownで書いて、ZIPで出せる。スマホでもPCでも、同じ体験で。

[行動の促し]
今すぐ無料で始める →
```

現状の課題：「あと、」という名前だけではツールとして伝わらない。タグラインをロゴとセットで見せることが必須。

#### noteとATOの導線設計

```
noteで記事を書く人
    ↓
note記事（開発日記）を読む
    ↓
LP（3行で何かを理解する）
    ↓
ATOを試す（初回起動 → サンプルプロジェクトが開く）
    ↓
「下書きをATOで書いて、noteに貼る」ワークフローを習慣化
    ↓
友人・フォロワーへの紹介
```

#### 初回オンボーディングの設計方針

「認知負荷ゼロ」は「何も表示しない」ことではなく「迷わせないこと」。

```
初回起動時：
→ サンプルプロジェクトが開いた状態でスタート
→ 「まずここをタップしてみてください」のツールチップを1つだけ表示
→ 隠れた操作（右半分長押しなど）は最初に一度だけ教える

2回目以降：
→ 通常のシンプルなUIに戻る
```

### Phase 3：定着（口コミを生む）

**目標**：使い続けてもらい、自然に紹介してもらう

#### 口コミが起きやすい設計

- **ZIPエクスポート** → 「データを自分で持てる」体験をした人が周りに話す
- **スマホ→PC連携** → 「移動中に書いたものが家のPCで続きから書けた」体験をした人が話す
- **和のUI** → スクリーンショットをSNSで共有したくなる美しさ

#### 定着を高める施策

| 施策 | 内容 |
|---|---|
| 使い方ガイドのnote記事 | 「noteとATOの組み合わせ方」という具体的なワークフロー紹介 |
| ユーザーの声を集める | 試験公開中のフィードバックを積極的に取得・公開（透明性） |
| ロードマップの公開 | 「次はAI機能を実装中」を定期的に発信し、期待を持続させる |

---

### Phase 4：収益化（プレミアム展開）

**目標**：持続可能な運営基盤を作る

#### 無料/有料の境界線（案）

| 機能 | 無料 | 有料（プレミアム） |
|---|---|---|
| プロジェクト数 | 3まで | 無制限 |
| ストレージ | 基本容量 | 拡張容量 |
| AIエージェント | なし | あり（将来） |
| バージョン管理 | なし | あり（将来） |
| エクスポート | ZIP対応 | 同左（無料で維持） |

#### noteコミュニティの課金文化

noteユーザーはサポート・メンバーシップ文化で課金慣れしている。「機能のために払う」だけでなく「作者を応援したい」課金も起きやすい。開発日記で人間性を出しておくことが、将来の課金転換率に直結する。

## 補足：ターゲットを絞ることの重要性

4つのセグメント（思考駆動型・作品駆動型・内省駆動型・集中駆動型）はいずれもATOに合致するが、**初期は「noteで書く思考駆動型」に絞ること**を推奨する。

理由：
1. すでに自分たちがそのコミュニティの中にいる（原点のTwitterポストがそこから来ている）
2. noteというプラットフォームを通じたコンテンツマーケティングが最も低コストで効く
3. noteユーザーは課金文化があり、将来の収益化への道が開けている
4. 「noteの下書きツール」というポジションが明確で、競合が存在しない

全員に刺さろうとすると、誰にも刺さらない。最初の100人をnote経由で作ることが、その後の口コミグロースへの最短経路。
