# SUMMARY

## 情報の優先度や責任範囲

ドキュメントと実装の間で矛盾が生じた場合、以下の優先度で正となる情報を判断する。

### 上流（What: 何を作るか）

| 文書 | 範囲 |
| --- | --- |
| 00_プロダクトデザイン | プロダクトの思想・コンセプト・UI指針 |
| 01_要件定義 | 機能・非機能要件（顧客との合意事項） |

### 中流（How: どう作るか）

| 文書 | 範囲 |
| --- | --- |
| 02_アーキテクチャ設計 | システム構成・データフロー・モジュール責務 |
| 03_データ設計 | DB・ローカルストレージ・RLS |
| 04_API設計 | エンドポイント・リクエスト/レスポンス仕様 |
| 05_画面設計 | 画面レイアウト・UIコンポーネント・状態遷移 |
| 06_テスト設計 | テスト戦略・テスト項目・合格基準 |
| 07_セキュリティ設計 | 認証/認可・脅威対策・監査ログ |
| 08_運用デプロイ設計 | 環境構成・CI/CD・監視 |

### 下流（実装）

| 対象 | 範囲 |
| --- | --- |
| ソースコード | 中流の設計に従った実装 |
| テストコード | テスト設計に従ったテスト実装 |
| インフラ設定 | 運用デプロイ設計に従った環境構築 |

## 作業別マップ

| 作業内容 | 更新対象ドキュメント |
| --- | --- |
| UIデザイン変更（カラー、フォント、アニメーション等） | 00_プロダクトデザイン |
| 新機能追加・仕様変更 | 01_要件定義 |
| モジュール構成・データフロー変更 | 02_アーキテクチャ設計 |
| テーブル定義・ローカルストレージ変更 | 03_データ設計 |
| APIエンドポイント追加・変更 | 04_API設計 |
| 画面レイアウト・UIコンポーネント変更 | 05_画面設計 |
| テストケース追加・テスト方針変更 | 06_テスト設計 |
| 認証・セキュリティ対策変更 | 07_セキュリティ設計 |
| デプロイ・環境設定変更 | 08_運用デプロイ設計 |
| 新機能追加（フルスタック） | 01_要件定義 → 02_アーキテクチャ設計 → 03_データ設計 → 04_API設計 → 05_画面設計 → 06_テスト設計 |
| 認証方式変更 | 07_セキュリティ設計 → 04_API設計 → 02_アーキテクチャ設計 |
| データモデル変更 | 03_データ設計 → 04_API設計 → 02_アーキテクチャ設計 |
