# データ設計

## サマリ

このドキュメントでは、用語/ドメイン定義、データモデル（テーブル一覧・定義・file_id発行ルール）、ローカルストレージ、RLSポリシーを定める。DBはCRDTスナップショット中心で、ファイル/フォルダ構造と内容はCRDTドキュメントに集約。

## 本文

### 用語/ドメイン定義

| 用語 | 定義 | 備考 |
|------|------|------|
| User | アプリケーションを利用する認証済みユーザー | - |
| Project | ファイルとフォルダを束ねる作業単位 | ユーザーごとに複数所有可能 |
| Folder | ファイルを階層的に整理するためのコンテナ | 構造はCRDTドキュメントで表現 |
| File | マークダウンをはじめとするファイル | 内容はCRDTドキュメントで表現 |

### データモデル

#### Postgres

##### テーブル一覧

| テーブル名 | 説明 |
|-----------|------|
| users | ユーザー情報 |
| projects | プロジェクト情報 |
| file_tree_docs | プロジェクト単位のファイルツリーCRDTスナップショット |
| file_content_docs | ファイル単位の内容CRDTスナップショット |

##### テーブル定義

###### users

| カラム名 | 型 | 説明 |
|----------|-----|------|
| id | UUID | PK |
| google_id | TEXT | Google OAuth ID |
| email | TEXT | メールアドレス |
| name | TEXT | 表示名 |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

###### projects

| カラム名 | 型 | 説明 |
|----------|-----|------|
| id | UUID | PK |
| owner_id | UUID | FK → users.id |
| name | TEXT | プロジェクト名 |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

###### file_tree_docs

| カラム名 | 型 | 説明 |
|----------|-----|------|
| project_id | UUID | PK, FK → projects.id |
| doc | BYTEA | YjsドキュメントのUint8Array |
| updated_at | TIMESTAMP | 最終更新日時 |

###### file_content_docs

| カラム名 | 型 | 説明 |
|----------|-----|------|
| file_id | UUID | PK（file_tree_docs内のCRDTで生成） |
| project_id | UUID | FK → projects.id（RLS用） |
| doc | BYTEA | YjsドキュメントのUint8Array |
| updated_at | TIMESTAMP | 最終更新日時 |

#### 補足

- `file_id`はクライアント側でUUID生成し、`file_tree_docs`のCRDT内に記録（オフライン対応のため）
- ファイル作成時、同じ`file_id`で`file_content_docs`レコードも作成

#### ローカルストレージ

| キー | 型 | 説明 |
|------|-----|------|
| `ato:file_tree_doc:{projectId}` | string (Base64) | ファイルツリーCRDTスナップショット |
| `ato:file_content_doc:{fileId}` | string (Base64) | ファイル内容CRDTスナップショット |
| `ato:settings` | string (JSON) | ユーザー設定 |
| `ato:current_project_id` | string | 選択中プロジェクトID |
| `ato:current_file_id` | string | 選択中ファイルID |

settingsのスキーマ

```typescript
type Settings = {
  font: 'bizud-gothic' | 'shippori-mincho'
  fontSize: number        // 16以上
  lineWrap: boolean
}
```

※容量制限・超過時の挙動については`要件定義.md`を参照

### セキュリティ

#### RLSポリシー

| テーブル | 操作 | ポリシー |
|----------|------|----------|
| projects | ALL | `owner_id = current_user_id()` |
| file_tree_docs | ALL | `project_id IN (SELECT id FROM projects WHERE owner_id = current_user_id())` |
| file_content_docs | ALL | `project_id IN (SELECT id FROM projects WHERE owner_id = current_user_id())` |

- `current_user_id()`はJWTから取得したユーザーIDを返す関数（API層で設定）

※セキュリティ要件については`要件定義.md`を参照
