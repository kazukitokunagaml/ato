# プロダクトデザイン

## サマリ

このドキュメントでは、思想、問題、価値、ターゲット、UI（カラー/スペーシング/タイポグラフィ/アニメーション/コンポーネント/アイコン）を定める。「無為自然」をコンセプトに、4つの壁（場所・疲れ・サービス・機能）を解消し「紙とペンのような自然な書き心地」を実現する。

## 本文

### 思想

情報通信が生まれ、インターネットが敷かれ、SNSが生まれた現代。僕らは文字を書くことをやめて、文字を打つようになった。誰もが思想やアイデアを、全世界に向けて瞬間で伝えることができるようになった。

文字を書くなら、紙とペンがあれば、山の中でも、街の中でも、どこでも同じように、ごく自然に書ける。

だが、文字を打つとなるとそうはいかない。大抵のアプリはPC版かスマホ版しかなく、たとえば移動中には作業できない。それにアプリのデータを外に出すのも大変だ。急速に進む電子化の裏で、かつて紙の時代にあった「自然な書き心地」が、静かに失われている。

人は、文字を使って、思いや知識を何千年先の世界に伝えてきた。人にとって、「書く」ことは生物としての個性であり、とても自然な行為だ。「書く」ことは自然であるべきだ。

心が動いた時。アイデアを閃いた時。その時、その場所でしか得られない体験を、そのまま文字に残せるような道具を作りたい。そう思った。

### 問題

現代の「書く」作業は、不自由だと思う。既存のアプリは、「不自由の壁」に縛られ、自然な書きごごちが失われていると思う。

**場所の壁**: スマホ版だと機能が制限されたり、そもそもスマホ版がなかったり、場所が変わるたびに作業や思考が途切れてしまう。
**疲れるという壁**: 使いこなすのに勉強が必要だったり、煩雑でどこに何があるかわからなくなったり、表示が多すぎて気が散ったり、心理的に疲れてしまう。
**サービスの壁**: アプリ内で閉じていて外部連携がしづらく、簡単に他のアプリに変えたり、元のアプリに戻ったりできない。紙のノートを持ち歩くように、データを自由に動かしたい。
**機能の壁**: 版管理やAIエージェントのような、執筆を便利にする最先端の機能を使うために、IT技術を勉強しなければならない。

- VSCode
  - 👍 GitやAIエージェントなど、機能性と拡張性が高い
  - 👎 スマホに最適化されておらず、執筆に不要な機能が多く気が散る
- Obsidian
  - 👍 スマホ対応があり、mdファイルで外部連携が可能
  - 👎 学習コストが高く、UIが複雑で認知負荷も高い
- Notion
  - 👍 デザインが優れ、認知負荷が低く、すぐに使いこなせる
  - 👎 独自形式でAI連携が弱いほか、エクスポートが大変
- Google Keep
  - 👍 PCでもスマホでも即座に書ける
  - 👎 機能が限定的で、断片的なメモの溜まり場になる
- iA Writer
  - 👍 書くことに集中でき、疲れない
  - 👎 OS別課金が高額。スマホではAI使えない
- Nola
  - 👍 小説特化で素人でも触れる
  - 👎 ドメイン特化で、汎用的なエディターではない
- しずかなインターネット・いろはエディター
  - 👍 感情に訴えかけるUIUXで、書くことが楽しくなる
  - 👎 機能が限定的で、AIエージェントなどは使えない

### 価値

- コンセプト: 「無為自然」
- 名称: 「道教思想 > 道 > TAO > ATO > あと、」
- 価値基準: 「自然さを損なう不自由さはないか？」
- キャッチコピー: 「只々、自然に書くためのアプリ。」

**場所の壁**: 移動時もすぐに作業を再開できる、作業内容のデバイス共有機能
**疲れるという壁**: すぐに使いこなせるほど直感的で、作業を邪魔しないUI
**サービスの壁**: いつでも移行でき、色々連携できる、標準技術とエクスポート機能
**機能の壁**: 版管理やAIエージェントを始めとする拡張機能と、それらを知識なしで使えるUI

### ターゲット

「書く」スタイルが異なる、4つのセグメントをターゲットとする。()は各セグメントに刺さる要素。

**思考駆動型**: 思いついたらすぐ書き残したいタイプで、いつもどこかに思想や哲学をメモしている。思考を中断させられることが嫌いで、ノマドでユビキタスな機能が好き。(場所の壁・疲れるという壁)

**作品駆動型**: プロダクトそのものに強いこだわりがある。考え尽くされたUIUXや、目新しい機能性を面白いと感じる。(疲れるという壁・サービスの壁・機能の壁)

**内省駆動型**: 成長志向で、とにかくレポートなどの成果物を出そうとする。反省や壁打ちによって、自分の成長を実感するのが好き。(機能の壁)

**集中駆動型**: 一切邪魔されることなく作業したい職人気質で、自分の仕事道具を厳選する。自分で細かく調整できるカスタム性が好き。（疲れるという壁・サービスの壁・機能の壁）

### UI

#### 基本原則

- **「無為自然」**: 「読ませない・探させない」を原則とする
- **和の自然さ**: 道教思想と和の文化をプロダクトの個性とする
- **認知負荷ゼロ**: 無駄な表示や機能の削除、ユーザーアクション時のガイド表示、使いこなすまでの学習コストを最小化など

#### カラーパレット

##### ルール

- テキスト: 背景に対して`4.5:1`以上
- 非テキスト(アイコン/ボーダー): 背景に対して`3:1`以上
- 注意: 薄墨(#bdbebe)はテキストに使用しない

##### カラー

- アプリ背景/文字色: `--color-white`: #fffffe; /* 白 */
- アプリ背景/文字色: `--color-black`: #373737; /* 墨 */
- UI背景: `--color-light-gray`: #e4e5e6; /* 白金 */
- UI背景: `--color-dark-gray`: #bdbebe; /* 薄墨 */
- LINK: `--color-blue`: #38a1db; /* 露草 */
- SUCCESS: `--color-green`: #007e66; /* 翠 */
- WARNING: `--color-yellow`: #f8b500; /* 山吹 */
- ERROR: `--color-red`: #e94824; /* 京緋 */

#### スペーシング

- 基準単位
  - `8px`
- スケール
  - `8の倍数`で統一（`8px` / `16px` / `24px `/ `32px` / `64px`）
- 原則
  - 一貫性: 同じ要素には同じ余白
  - 階層: 重要なものほど余白大
  - 関係性: 近いものは関連、離れたものは独立

#### レイアウト

- ブレイクポイント
  - `768px`

#### タイポグラフィ

- フォント
  - `BIZ UDゴシック`: デフォルト。ユニバーサルデザインで見やすさ優先。
  - `しっぽり明朝`: 情緒的なフォント。ユーザーカスタム可能。
  - `Noto_Sans_Mono`: コードブロック用。等幅フォント。
- ウェイト
  - `N(400)`と`B(700)`の2段階。ただしコードブロックフォントは`N`のみ。
- サイズ
  - 入力領域は`16px`以上(ブラウザのAutoFocus回避)
- 行間
  - 本文: `160%`(ゆとりをもって見やすく)
  - 見出し: `140%`(まとまりを持たせる)

#### アニメーション

- トランジション
  - `200〜300ms`(ユーザーを焦らせない)
- モーダル表示
  - 背景をすりガラスの様にぼかして、他要素の操作不可に
- フェードイン/アウト
  - 透明度を変化させながらスライド
- アクセシビリティ
  - `prefers-reduced-motion`でアニメーション軽減可能

#### コンポーネント

- 形状
  - 角丸`border-radius: 8px`
- アクセシビリティ
  - ユーザーでカスタム可能
  - 利き手によってサイドバー系の位置を変更

#### アイコン

- デザイン
  - MUI3で統一
- サイズ
  - タッチターゲットは最低44px × 44px確保
- 余白
  - アイコン自体余白はゼロ
- ガイドライン
  - ユーザーに何かの行動を促すテキストには必ずアイコンを添える
